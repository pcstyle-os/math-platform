# design-system

## Description

Manage tokens, component variants, and theming (e.g., for Radix UI, shadcn/ui).

## Instructions

- Define a central theme config (colors, spacing, fonts).
- Build components using atomic design principles (atoms, molecules, organisms).
- Use tools like `class-variance-authority` (CVA) to manage component variants.
- Ensure all components support dark mode and custom primary colors.
- Document components in a Storybook or equivalent if applicable.
